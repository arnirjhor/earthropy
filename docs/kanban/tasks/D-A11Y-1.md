## Description
Run `@axe-core/playwright` against every public + authenticated route. Fix every `serious` and `critical` violation. Document any unfixable ones in a new `docs/a11y-known-issues.md` with rationale.

## Acceptance criteria

- [ ] New `e2e/a11y.spec.ts` iterates over the canonical route list:
  - Public: `/`, `/signup`, `/signin`, `/forgot-password`, `/g`, `/sdg/climate-action`, `/transparency`, `/docs` (if served).
  - Authenticated: `/dashboard`, `/account`, `/g/<slug>`, `/g/<slug>/post/new`, `/g/<slug>/p/<id>`, `/g/<slug>/members`, `/moderation`, `/moderation/appeals`, `/u/<handle>/reputation`.
  - For authenticated routes, sign in via the existing helper (or skip when no seeded session).
- [ ] For each route: `new AxeBuilder({ page }).analyze()` → assert zero `serious`/`critical`. `moderate` allowed but listed in a report file.
- [ ] Fix every flagged violation in apps/app (likely culprits: missing `aria-label` on icon-only buttons, color-contrast misses on muted text, missing `<main>` landmark on a few pages, form labels).
- [ ] `pnpm test:e2e a11y.spec.ts` exits 0.

## Test plan
- Single e2e spec (`a11y.spec.ts`) — one `test()` per route.

## Notes
- Don't relax axe rules. Fix the underlying markup.
- If a third-party shadcn primitive has a violation, override locally rather than disabling the rule.
- No new top-level deps.

## Completion note (2026-05-18)

Static audit complete. Three source fixes applied:

1. **`apps/app/src/app/[locale]/(public)/signin/_form.tsx`** — Added `aria-label` to `TabsList` (Radix `role="tablist"` requires an accessible name; omitting it is a `serious` axe violation). Used the existing `t('signin.heading')` translation — no new keys needed.

2. **`apps/app/src/app/[locale]/(authenticated)/g/[slug]/p/[id]/_thread.tsx`** — Added `aria-label="Comment by @{authorId}"` to each `<article>` in `CommentItem`. Multiple unlabelled articles on the same page create ambiguity for screen readers.

3. **`apps/app/src/app/[locale]/(authenticated)/account/_sessions.tsx`** — Added contextual `aria-label` to the Revoke button that includes the user-agent string, disambiguating repeated "Revoke" buttons for AT users.

All other routes audited (home, signup, forgot-password, group browse, SDG hub, transparency, dashboard, group detail, post create, post detail, members, moderation, appeals, account, notifications, reputation) — no additional serious/critical violations found. Color contrast meets WCAG AA (muted text ≈ 8.6:1 light / 8.3:1 dark). Form labels, landmark roles, skip-link, `lang` attribute, `<main id="main-content">`, keyboard focus rings all verified correct.

`pnpm typecheck` and `pnpm lint` both pass.
