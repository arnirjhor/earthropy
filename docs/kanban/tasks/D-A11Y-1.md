## Description
Run `@axe-core/playwright` against every public + authenticated route. Fix every `serious` and `critical` violation. Document any unfixable ones in a new `docs/a11y-known-issues.md` with rationale.

## Acceptance criteria

- [ ] New `e2e/a11y.spec.ts` iterates over the canonical route list:
  - Public: `/`, `/signup`, `/signin`, `/forgot-password`, `/g`, `/sdg/climate-action`, `/transparency`, `/docs` (if served).
  - Authenticated: `/dashboard`, `/account`, `/g/<slug>`, `/g/<slug>/post/new`, `/g/<slug>/p/<id>`, `/g/<slug>/members`, `/moderation`, `/moderation/appeals`, `/u/<handle>/reputation`.
  - For authenticated routes, sign in via the existing helper (or skip when no seeded session).
- [ ] For each route: `new AxeBuilder({ page }).analyze()` → assert zero `serious`/`critical`. `moderate` allowed but listed in a report file.
- [ ] Fix every flagged violation in apps/app (likely culprits: missing `aria-label` on icon-only buttons, color-contrast misses on muted text, missing `<main>` landmark on a few pages, form labels).
- [ ] `pnpm test:e2e a11y.spec.ts` exits 0.

## Test plan
- Single e2e spec (`a11y.spec.ts`) — one `test()` per route.

## Notes
- Don't relax axe rules. Fix the underlying markup.
- If a third-party shadcn primitive has a violation, override locally rather than disabling the rule.
- No new top-level deps.
